# 專案管理大師指令 (Project Management Master Prompt)

如果你要開啟一個新的專案，請將以下框框內的內容複製並傳送給 AI 助手。這會自動建立一套包含 GitHub 同步、開發日誌、教學文件與自動化腳本的專業工作流。

---

```markdown
## 指令名稱：專業工作流初始化與專案管理

**執行任務：**
1. **建立儲存庫**：在當前目錄下建立一個專案資料夾，並根據我的要求在 GitHub 上建立同名的 Public Repo 並完成關聯。
2. **建立自動化腳本**：
   - **建立 `startup.sh`**：功能包含自動執行 `git pull`、顯示最新的「開發日誌」內容，並向 AI 助手回報目前進度。
   - **建立 `ending.sh`**：功能包含自動執行 `reconstruct_dialog.py` 以還原完整對話、提醒 AI 助手更新當天日誌與教學文件，然後自動執行 `git add / commit / push`。
   - **建立 `reconstruct_dialog.py`**：這是一個關鍵腳本，讓 AI 助手能讀取系統底層的 `overview.txt`，確保即使對話再長、歷史紀錄被壓縮，也能將所有原始對話備份到 `dialog.md` 中。
3. **建立文件系統**：
   - **開發日誌**：建立 `YYYY.MM.DD開發日誌.md`，記錄所有「有意義」的對話、除錯過程與技術結論。
   - **教學目錄 `/Tutorial`**：建立此資料夾，並撰寫 `Tutorial_1.md` 開始記錄第一項工作，後續教學需邏輯連貫並依序編號。
4. **GitHub 管理規範**：所有具備技術意義的進度更新（如工作流修改、程式碼優化）都必須在收工前透過 `ending.sh` 邏輯同步至 GitHub。

**現在請開始詢問我這次新專案的名稱與 GitHub 帳號資訊。**
```
